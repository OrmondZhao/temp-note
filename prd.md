# 临时笔记网页应用 PRD

## 1. 文档信息
- 产品名称：临时笔记
- 文档类型：产品需求文档（PRD）
- 版本：V1.1
- 状态：已同步原型
- 目标：定义一个可在浏览器中随时打开、用于临时收集和查看文本与图片素材的轻量网页应用

## 2. 背景与问题
在日常工作中，用户经常会临时复制一段文字、截取一张图片、保存一个链接或记录一个灵感。此类信息通常“即来即用”，但现有工具往往存在以下问题：
- 信息分散在聊天软件、桌面文件、截图目录或临时便签中，不易统一管理。
- 常规笔记工具偏重长期整理，不够适合快速收集和快速回看。
- 许多临时素材缺乏统一入口，导致后续查找成本高、容易遗失。

因此需要一个轻量级网页工作台，专门承接这些“临时但重要”的素材。

## 3. 产品目标
### 3.1 产品定位
一个打开即用的浏览器网页页面，作为个人工作中的临时素材收纳板。当前原型采用“中间主编辑区 + 右侧快速查找区 + 左侧全局工具区”的工作台结构。

### 3.2 核心目标
- 让用户快速记录文本、图片等即时素材。
- 让用户能够随时打开并查看历史素材。
- 让素材在本地被可靠保存，不因刷新页面而丢失。
- 让单条素材可独立导出、导入和复用。

### 3.3 成功标准
- 用户可以在 10 秒内完成首次记录。
- 用户刷新页面后，内容仍可恢复。
- 用户能够通过搜索快速找回已保存内容。

## 4. 用户画像
### 4.1 核心用户
- 需要高频收集临时信息的个人办公用户。
- 经常在多任务之间切换，希望随手记录工作素材的人。
- 不想使用复杂知识管理工具的轻量用户。

### 4.2 典型需求
- 临时保存文案、摘录、备注、待办。
- 临时保存截图、参考图、设计截图。
- 单条素材单独备份、迁移、分享。
- 快速回看之前收集过的素材。

## 5. 使用场景
- 用户在浏览网页时复制一段文字，立即粘贴到页面保存。
- 用户截屏后直接粘贴图片到页面，作为后续参考。
- 用户拖入一张图片文件，快速归档为参考素材。
- 用户在处理任务前打开页面，查看此前收集的信息。

## 6. 产品范围
### 6.1 V1 必做
- 文本素材创建、编辑、删除。
- 图片素材创建、查看、删除。
- 图片以内嵌 Base64 形式直接写入正文，无需独立附件管理。
- 自动保存到本地（IndexedDB + storage.persist 权限申请）。
- 搜索与基础筛选。
- 置顶或收藏。
- 单条素材导入与导出。
- 浏览器中直接访问使用。

### 6.2 V1 可延后
- 多端同步。
- 账号体系。
- OCR 识别图片文字。
- 文件附件管理。
- 协作分享。

## 7. 核心流程
### 7.1 新增素材流程
1. 用户打开页面。
2. 用户在中间主编辑区输入标题、正文和标签。
3. 用户粘贴截图或拖入图片文件，图片直接以内嵌 Base64 形式写入正文。
4. 用户点击保存后，系统生成或更新当前素材并自动保存。
5. 用户可随时点击"放弃"按钮丢弃未保存的草稿。

### 7.2 查找素材流程
1. 用户输入关键词或选择筛选条件。
2. 系统即时过滤卡片列表。
3. 用户点击目标卡片后，主编辑区切换到对应素材。

### 7.3 备份恢复流程
1. 用户导出当前素材数据。
2. 在需要时导入该 JSON 文件。
3. 系统恢复为单条素材；旧版整包 JSON 仍兼容导入。

## 8. 功能需求
### 8.1 页面入口
优先级：P0
- 页面可通过浏览器直接打开。
- 默认进入主工作台，无需登录即可使用。
- 页面加载后应立即可进行新增或查看操作。

### 8.2 文本素材
优先级：P0
- 支持新建文本卡片。
- 支持编辑标题、正文、标签、备注。
- 支持删除文本卡片。
- 支持展开查看完整正文。
- 正文编辑区应作为主工作区的核心，尽量提供较大输入空间。

### 8.3 图片素材
优先级：P0
- 支持粘贴图片直接写入正文。
- 支持拖拽上传图片文件写入正文。
- 支持点击正文内的图片查看原图。
- 图片以内嵌 Base64 形式存储在正文中，不再有独立附件概念。

### 8.4 内容展示
优先级：P0
- 所有素材以卡片形式展示。
- 默认按最近更新时间或创建时间排序。
- 文本卡片展示标题与正文纯文本摘要（HTML 标签不显示）。
- 含图片的卡片在摘要中显示"《图片》"标识，正文保留完整富文本。

### 8.5 搜索与筛选
优先级：P0
- 支持关键词搜索标题、正文、标签。
- 支持按类型筛选文本或图片。
- 支持按置顶、收藏进行快速筛选。

### 8.6 置顶与收藏
优先级：P0
- 支持将单条素材置顶。
- 支持收藏重要素材。
- 置顶内容在列表中优先显示。

### 8.7 自动保存
优先级：P0
- 所有新增、编辑、删除操作应自动保存。
- 用户刷新页面后数据应可恢复。
- 应提供明显的保存状态反馈。
- 保存介质为浏览器本地 IndexedDB。

### 8.8 导入与导出
优先级：P0
- 默认支持导出当前素材或当前草稿为单条 JSON。
- 支持导入单条 JSON 并作为新素材加入。
- 兼容旧版整包 JSON 导入。
- 导入失败时需提示原因，且不得影响已有数据。

### 8.9 排序管理
优先级：P1
- 支持手动调整卡片顺序。
- 或提供排序方式切换能力，例如按时间、按收藏、按置顶。

### 8.10 快捷操作
优先级：P1
- 支持常用快捷键，例如快速新建、快速搜索。
- 支持高频输入场景下的低成本操作路径。

## 9. 交互设计原则
- 一切围绕“快速收集”和“快速回看”展开。
- 常用操作应尽量在 1 到 2 次操作内完成。
- 页面不应被复杂菜单打断主流程。
- 保存结果和操作状态需要有清晰反馈。
- 内容查看与编辑的入口要明确，避免误操作。

## 10. 数据结构建议
每条素材建议包含以下字段：
- `id`
- `type`（固定为 "text"，不再区分 "image" 类型）
- `title`
- `body`（图片以内嵌 `<img src="data:...">` 形式存在，富文本 HTML）
- `tags`
- `pinned`
- `favorite`
- `createdAt`
- `updatedAt`
- `sortOrder`

> 不再维护独立的 `attachments[]` 数组，所有图片以内联方式存在 body HTML 中。

## 11. 非功能需求
### 11.1 性能
- 页面首次进入后应能快速使用。
- 搜索、打开卡片、预览图片应保持流畅。

### 11.2 兼容性
- 优先兼容现代桌面浏览器。
- 在常见分辨率下应有良好的布局表现。
- 后续可扩展到移动端，但不作为 V1 主目标。

### 11.3 稳定性
- 刷新页面后数据不丢失。
- 导入失败不能覆盖已有数据。
- 图片和文本都应具备基本容错处理。
- 点击图片应能查看原图预览，不应只展示裁切缩略图。
- 浏览器启动时申请 `navigator.storage.persist()` 权限，授权后浏览器自动清理模式不会删除 IndexedDB 数据（用户手动清除"Cookies 和网站数据"除外）。

### 11.4 易用性
- 页面风格应偏工具属性，简洁克制。
- 信息层级清晰，避免视觉噪音。
- 用户无需学习成本即可快速上手。

## 12. 埋点与指标
### 12.1 核心指标
- 首次记录成功率
- 日活跃使用次数
- 平均素材创建耗时
- 搜索命中率
- 导入导出使用率

### 12.2 观察目标
- 用户是否会高频使用快速记录入口。
- 用户是否依赖搜索找回历史素材。
- 用户是否有导出备份的习惯。

## 13. 验收标准
- 页面可在浏览器中直接打开并使用。
- 可新增、编辑、删除文本素材。
- 可粘贴或上传图片，图片以内嵌 Base64 形式保存在正文中。
- 刷新后已保存内容可恢复。
- 可搜索、筛选、查看历史素材。
- 可导出并导入单条素材 JSON，且恢复结果正确。
- 新建素材时输入内容后切换到其他卡片，系统询问是否保存未内容。
- 草稿状态下可点击"放弃"按钮丢弃未保存内容。
- 复制功能支持将富文本 HTML 和纯文本同时写入剪贴板。
- 左侧统计区的"图片"计数为含图片的素材数量（body 含 `<img>` 标签）。

## 14. 风险与约束
- 仅使用本地存储时，无法天然实现跨设备同步。
- 图片体积较大时，本地存储容量与性能需要重点评估。
- 若后续引入登录和云端同步，系统复杂度会明显提升。
